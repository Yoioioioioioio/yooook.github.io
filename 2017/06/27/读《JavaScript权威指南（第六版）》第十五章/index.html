读《JavaScript权威指南（第六版）》第十五章

发表于 2017-06-27 | 分类于 JS

~关于DOM：html的节点树，document节点=>树的根部代表整个文档，text节点=>文本节点，elemet是html元素的节点。他们都是Node的子类。
~在低版本浏览器中getElementById匹配的Id不区分大小写，而且也可以返回匹配name
~html的name属性最初打算为了表单元素分配名字，和id不同的是name属性只有在极少的html元素中有效，包括表单，表单元素，<\iframe>,<\img>
~基于名字选择html元素，getElementsByName()
~document.all[]获取所有文档中的元素，现已经废弃
~element对象的children类似于childnodes但并不是标准属性
~对于类似<%i>,<%code>,<%span>这些内联元素，clientWidth和clientHeight总是返回0；
~<%form>元素中的元素都有一个form属性引用了该包含的表单，事件处理程序总能够通过this.form来得到form对象的引用，这样就意味着，表单的元素可以通过this.form.x得方式获取；
~在H5中，placeholder属性制定了用户输入前在输入域中的信息提示
~option对象可以通过options.length = 0l来移除所有选项，通过option[]获取某个option对象，具体插入删除用，insertBefore(),removeChild()方法
~在设置了defer或者async属性的脚本中不要使用document.write();
该章节内容广度大，但是就项目开发而言，实际运用涉及的少

如果有用，请留言点赞，或者提出你们的问题，共同学习！
